# Manual Drift Review

Use this skill when the task asks whether DayOS matches `DayOS_ClaudeCode_Manual.md` or `agentlayer-plan.md`.

## Procedure

1. Isolate the requested manual section instead of comparing the whole document blindly.
2. Compare it against live code in the owning repo.
3. Separate implemented behavior, partial scaffolding, and unimplemented roadmap items.
4. Cite the files that prove the current state.

## Return

- A clear drift summary.
- The code evidence for each conclusion.
- Any important ambiguities that still need manual verification.
