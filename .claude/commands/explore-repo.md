# explore-repo

Use `explore-agent` for broad read-only discovery when the request is not yet narrow.

Workflow:
1. Confirm whether the request targets the top-level workspace, `mobile/`, or both.
2. Explore current code before leaning on the manuals.
3. Summarize the relevant files, current behavior, and any roadmap drift.

Output:
- repo scope used
- relevant files and services
- implementation status and open questions
