# implement-mobile-feature

Use this when the task changes app behavior in `mobile/`.

Workflow:
1. Start from the relevant screen, service, store, or repository in `mobile/`.
2. Trace all impacted layers before editing.
3. Keep DB mappings, store state, services, and UI aligned.
4. Validate from the `mobile/` cwd.

Output:
- affected layers
- edits made or planned
- validation steps and remaining gaps
