# review-manual-drift

Use this when asked whether the repo matches the spec.

Workflow:
1. Identify the manual section or roadmap claim under review.
2. Inspect the live code in `mobile/` that would prove or disprove it.
3. Classify each claim as implemented, partial, scaffolded, or missing.

Output:
- manual claim
- proving files
- drift summary and implementation gaps
