# write-mobile-tests

Workflow:
1. Identify the exact behavior to prove.
2. Read the target service, screen, or store path first.
3. Add the narrowest useful test coverage the repo can support today.
4. Run the relevant validation command from `mobile/` when possible.

Output:
- files changed
- behavior covered
- remaining harness or coverage gaps
