# Mobile Tests

Use this skill for one focused app test file, targeted validation helper, or a testability review.

## Procedure

1. Scope the request to one behavior, service, or screen flow.
2. Prefer tests that prove real behavior over broad snapshots.
3. If the repo lacks a test harness for the requested slice, recommend the minimum viable validation path instead of forcing a weak test.
4. Keep the validation target anchored in current code paths.

## Return

- Files changed or proposed.
- Checks run and what they proved.
- Remaining harness or coverage gaps.
