# Release Check

Use this skill for app release-readiness, validation checklists, or repo-health checks.

## Procedure

1. Prefer concrete checks: Git status, package metadata, TypeScript validation, and native dependency review.
2. Keep blockers explicit instead of guessing.
3. Call out when manual device verification is still required.

## Return

- A checklist with pass, warn, or block outcomes.
- Commands run and what they proved.
- Remaining blockers or manual verification gaps.
